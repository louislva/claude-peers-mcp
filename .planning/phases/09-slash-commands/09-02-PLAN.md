<!-- Broker API patterns from cli.ts -->
From cli.ts:
- BROKER_URL: http://127.0.0.1:${CLAUDE_PEERS_PORT:-7899}
- /list-peers: POST with { scope: "machine", cwd: "/", git_root: null } returns Array of { id, pid, cwd, git_root, tty, summary, last_seen }
- /send-message: POST with { from_id: "cli", to_id, text } returns { ok: boolean, error?: string }
- /stats: GET returns { db_path, db_size_human, schema_version, retention: { messages_hours, sessions_days, waves_days }, counts: { peers, messages_total, messages_undelivered, ... } }

This command takes two arguments provided by the user: a peer ID and a message.

**Step 1 — Validate arguments:**
The user should provide the command as: `/comms-send <peer-id> <message>`

Run: `curl -s --max-time 3 -X POST -H 'Content-Type: application/json' -d '{"from_id":"cli","to_id":"PEER_ID","text":"MESSAGE"}' http://127.0.0.1:${CLAUDE_PEERS_PORT:-7899}/send-message`

Replace PEER_ID and MESSAGE with the user-provided values. Properly escape the message text for JSON.

If curl fails, print: `Broker is not running.`
If response has `ok: true`, print: `Message sent to <peer-id>.`
If response has `ok: false`, print: `Failed to send: <error>`
