- `let peerData: Peer[] = []` — cached peer list from last successful fetch
- `let scrollOffset: number = 0` — for vertical scrolling when peer list exceeds viewport

**render(startRow, startCol, width, height):**
1. Fetch peer data: `const data = await safeFetch<Peer[]>("/list-peers", { scope: "machine", cwd: "/", git_root: null })`
   - IMPORTANT: render() is synchronous per TabDef interface. Store data in module state. The actual fetch happens in start() and on each refresh timer tick. render() reads from `peerData`.
   - Actually, looking at gsd-watch.ts pattern: render() is synchronous and reads from module-level state. Data fetching happens separately and triggers re-render via lastRenderArgs pattern.

2. Strategy: Use a `fetchPeers()` async function that updates `peerData` and re-renders using lastRenderArgs pattern (same as gsd-watch.ts). Call fetchPeers() from start() and on refresh intervals. But wait — app.ts already calls render() on the refresh timer. So: make render() trigger an async fetch that updates state and re-renders. OR: simpler approach — since app.ts re-renders the active tab every REFRESH_MS, just do the async fetch at render time and cache results.

3. SIMPLEST APPROACH: Since render() is sync, use a separate async `fetchData()` that stores results in module state. The app.ts refresh timer calls render() every 2s. Inside render(), kick off a non-blocking fetchData() call (fire-and-forget) that updates peerData. On next render cycle, fresh data appears. First render shows "Loading..." or empty state.

4. Clear content area. For each peer in peerData (respecting scrollOffset):
   - Row format: `[ROLE] ID  PID:NNNNN  summary_text  last_seen_ago`
   - Role badge: Parse peer.summary for keywords — if contains "orchestrat" use badge("ORCH", C.purple), if contains "execut" use badge("EXEC", C.blue), if contains "proxy" or "decision" use badge("PROXY", C.yellow), else badge("PEER", C.gray)
   - PID: `fg(C.dimGray) + "PID:" + peer.pid`
   - Summary: `fg(C.text) + truncate(peer.summary, width/2)`
   - Last seen: compute seconds since peer.last_seen. Color: green (<30s), yellow (30-120s), red (>120s). Format as "Ns ago", "Nm ago", or "Nh ago".

5. If peerData is empty, show centered "No peers connected" in dimGray.

6. Footer line (last row of content area): `fg(C.dimGray) + BROKER_URL + " | " + peerData.length + " peer(s)"`

**handleKey(name):**
- "j" or "down": scrollOffset++ (clamped to peerData.length - viewportHeight)
- "k" or "up": scrollOffset-- (clamped to 0)
- Re-render after scroll via lastRenderArgs pattern

**start():** Call fetchData() immediately.
**stop():** Clear peerData, reset scrollOffset.

Per BRKR-01: Must show role badges, PID, summary, and color-coded last_seen.
