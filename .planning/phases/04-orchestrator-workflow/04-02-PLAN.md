**1. `export async function dispatchWave(myId: PeerId, gitRoot: string, waveNumber: number, phases: PhaseNode[], executors: AvailablePeer[]): Promise<{ waveId: number; assignments: Map<number, PeerId>; localPhases: PhaseNode[] }>`**
- Call `brokerFetch<{ wave_id: number; task_ids: number[] }>("/wave-create", { repo: gitRoot, phase: waveNumber, wave_number: waveNumber, tasks: phases.map(p => ({ name: \`phase-${p.number}\`, files: p.filesModified })) })`
- After wave-create, call `/wave-status` to check existing task states (per pitfall 4: idempotency on retry). Only dispatch tasks in "pending" state.
- For each pending task: if executors are available, pop one from the array and send `execute_phase` message via `brokerFetch("/send-message", { from_id: myId, to_id: executor.id, text: \`Execute phase ${phase.number}\`, msg_type: "execute_phase", payload: { phase_number, plan_path: \`.planning/phases/${phase.dir}/${padPhase(phase.number)}-PLAN.md\`, flags: "--no-transition --auto", wave_id, task_id, orchestrator_id: myId } satisfies ExecutePhasePayload })`
- NOTE: Do NOT call /task-start — the executor owns that transition (per pitfall 2 and research anti-pattern)
- If no executors available for a task, add the phase to `localPhases` array
- Return `{ waveId, assignments: Map<taskId, executorId>, localPhases }`
- Add internal helper `function padPhase(n: number): string { return String(n).padStart(2, "0"); }` (not exported)

**2. `export async function pollOrchestratorMessages(myId: PeerId): Promise<{ progresses: Array<{ msgId: number; payload: PhaseProgressPayload }>; completions: Array<{ msgId: number; payload: PhaseCompletePayload }>; blocks: Array<{ msgId: number; payload: PhaseBlockedPayload }>; statusResponses: Array<{ msgId: number; payload: StatusResponsePayload }> }>`**
- Call `brokerFetch<PollMessagesResponse>("/poll-messages", { id: myId })`
- Filter messages by msg_type: "phase_progress", "phase_complete", "phase_blocked", "status_response"
- Parse each message's payload field (JSON.parse)
- Return categorized arrays
- Do NOT ACK here — caller decides when to ACK (same pattern as proxy pollForChoices)

**3. `export async function ackMessages(messageIds: number[]): Promise<void>`**
- If messageIds is empty, return immediately
- Call `brokerFetch("/ack-message", { message_ids: messageIds })`

**4. `export async function sendStatusRequest(myId: PeerId, executorId: PeerId, taskId: number): Promise<void>`**
- Call `brokerFetch("/send-message", { from_id: myId, to_id: executorId, text: "Status check", msg_type: "status_request", payload: { task_id: taskId } })`

**5. `export async function reclaimExecutorTask(myId: PeerId, executorId: PeerId, taskId: number, waveId: number, reason: string): Promise<void>`**
- Call `brokerFetch("/send-message", { from_id: myId, to_id: executorId, text: "Reclaiming task", msg_type: "reclaim_task", payload: { task_id: taskId, wave_id: waveId, reason } })`
- Call `brokerFetch("/task-blocked", { task_id: taskId, error: reason })`

**6. `export function shouldDelegate(phase: PhaseNode, availableExecutorCount: number, runningFiles: string[], hasHumanCheckpoint: boolean): boolean`**
- Return false if `availableExecutorCount === 0`
- Return false if phase has fewer than 3 files in `filesModified` (small phase, not worth delegation overhead — heuristic for <3 tasks proxy)
- Return false if any of `phase.filesModified` overlaps with `runningFiles` (file conflict with in-progress work)
- Return false if `hasHumanCheckpoint` is true
- Otherwise return true

**3. `export async function waitForWaveComplete(myId: PeerId, waveId: number, assignments: Map<number, PeerId>): Promise<{ completed: PhaseCompletePayload[]; blocked: PhaseBlockedPayload[]; reclaimed: number[] }>`**
- Initialize: `progressTimestamps = new Map<number, number>()` (taskId -> last progress epoch), `statusRequestSent = new Map<number, number>()` (taskId -> when status_request was sent), `completed: PhaseCompletePayload[] = []`, `blocked: PhaseBlockedPayload[] = []`, `reclaimed: number[] = []`
- Loop:
  1. `await new Promise(r => setTimeout(r, 10_000))` — 10s poll interval (ORCH-07)
  2. Drain message queue FIRST (per pitfall 3): call `pollOrchestratorMessages(myId)`
     - For each progress: update `progressTimestamps.set(payload.task_id, Date.now())`, ACK the message
     - For each completion: add to `completed`, ACK the message
     - For each block: add to `blocked`, ACK the message
     - For each statusResponse: clear from `statusRequestSent`, update progressTimestamps, ACK the message
  3. Call `brokerFetch<{ wave: Wave; tasks: TaskAssignment[] }>("/wave-status", { wave_id: waveId })`
  4. If `wave.status === "completed" || wave.status === "failed"`, break
  5. For each running task: check `progressTimestamps` — if now - lastProgress > 120_000 (ORCH-08):
     - If `statusRequestSent` has this task and now - sentTime > 30_000: call `reclaimExecutorTask`, add taskId to `reclaimed`, clear from tracking
     - Else if not in `statusRequestSent`: call `sendStatusRequest`, record in `statusRequestSent`
- Return `{ completed, blocked, reclaimed }`
