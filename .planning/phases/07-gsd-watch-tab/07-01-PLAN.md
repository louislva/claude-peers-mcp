Create `tui/tabs/gsd-watch-parser.ts` exporting:

**Types:**
```typescript
type NodeStatus = "DONE" | "EXEC" | "PLAN" | "DISC" | "VRFY" | "PEND";
type NodeKind = "milestone" | "phase" | "plan";

interface TreeNode {
  kind: NodeKind;
  name: string;          // e.g. "v1.1 comms-watch TUI Dashboard", "Phase 7: GSD Watch Tab", "07-01-PLAN.md"
  status: NodeStatus;
  children: TreeNode[];
  expanded: boolean;     // default: true for milestones/phases, true for plans (leaf)
}

interface GsdTree {
  roots: TreeNode[];           // milestone nodes
  completedPlans: number;
  totalPlans: number;
}
```

**parseGsdTree(planningDir: string): GsdTree**
1. Read `{planningDir}/ROADMAP.md` using `Bun.file().text()`
2. Parse milestone headers (e.g. `## v1.0 ...`, `### v1.1 ...`) — look for lines matching `**v{X}.{Y} {name}**`
3. Parse phase entries under each milestone section:
   - Lines like `- [x] **Phase N: Name**` or `- [ ] **Phase N: Name**`
   - `[x]` = completed phase
4. For each phase, scan `{planningDir}/phases/{NN}-{slug}/` directory:
   - List files matching `*-PLAN.md`, `*-SUMMARY.md`, `*-VERIFICATION.md`, `*-CONTEXT.md`
   - Also parse sub-entries under the phase in ROADMAP (lines like `- [x] NN-MM-PLAN.md -- description`)
5. Derive plan status from file presence:
   - `[x]` in ROADMAP plan list -> DONE
   - `*-VERIFICATION.md` exists -> VRFY
   - `*-SUMMARY.md` exists -> EXEC (executing/executed)
   - `*-PLAN.md` exists -> PLAN
   - `*-CONTEXT.md` exists -> DISC (discussed)
   - Otherwise -> PEND
6. Derive phase status: if all plans DONE -> DONE; if any EXEC -> EXEC; if any PLAN -> PLAN; if any DISC -> DISC; else PEND. Completed phases ([x]) always DONE.
7. Count completedPlans (status=DONE) and totalPlans across all phases.

**watchPlanning(planningDir: string, onChange: () => void): () => void**
- Use `fs.watch(planningDir, { recursive: true })` to watch for file changes
- Debounce onChange calls (100ms) to avoid rapid re-parses
- Return a cleanup function that closes the watcher

**Tests** (`tui/tabs/gsd-watch-parser.test.ts`):

- Test parseGsdTree against the mock data
- Verify node counts, statuses, and progress calculation
- Test edge cases: empty phase dir, missing ROADMAP.md (return empty tree)
