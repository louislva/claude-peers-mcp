Replace `tui/tabs/gsd-watch.ts` entirely. Keep the same exports (TAB_NAME, REFRESH_MS, render, start, stop, handleKey).

**Module state:**
```typescript
let tree: GsdTree = { roots: [], completedPlans: 0, totalPlans: 0 };
let stopWatcher: (() => void) | null = null;
let cursorIndex: number = 0;   // index into visible (flattened) nodes
let scrollOffset: number = 0;  // first visible row in viewport
```

**Constants:**
```typescript
export const TAB_NAME = "GSD Watch";
export const REFRESH_MS = 0;  // event-driven, not polled
```

**start():**
1. Detect .planning/ directory — look for `.planning/ROADMAP.md` relative to process.cwd()
2. Call `tree = parseGsdTree(planningDir)` for initial data
3. Call `stopWatcher = watchPlanning(planningDir, () => { tree = parseGsdTree(planningDir); })` — the app.ts render loop will pick up changes on next render call. NOTE: The app needs to know to re-render. Since REFRESH_MS=0, the watcher callback should trigger a re-render. Import the app instance or use a flag that render() checks. Simplest approach: store a `dirty` flag, and have the watcher set `dirty=true`. But app.ts doesn't poll dirty flags for REFRESH_MS=0 tabs. Instead, write directly to stdout from the watcher callback — call the module's own render with cached dimensions. Store last render dimensions in module state.

**stop():**
1. If stopWatcher exists, call it
2. Reset state

**render(startRow, startCol, width, height):**
1. Store dimensions for watcher-triggered re-renders: `lastRenderArgs = { startRow, startCol, width, height }`
2. Flatten the tree into visible nodes (respecting expanded/collapsed state):
   - Walk tree.roots recursively
   - For each node, add to visible list with its depth level
   - If node.expanded=false, skip its children
3. Calculate viewport: `visibleNodes[scrollOffset..scrollOffset+viewportHeight]` where viewportHeight = height - 2 (reserve 1 line for progress bar, 1 for spacing)
4. For each visible node, render one line:
   - Indent: `"  ".repeat(depth)` — 2 spaces per level
   - Connector: For milestones none, for phases `|-- `, for plans `   |-- `
   - Collapse indicator: `v ` if expanded with children, `> ` if collapsed with children, `  ` if leaf
   - Status badge using render.ts badge():
     - DONE -> badge("DONE", C.green)
     - EXEC -> badge("EXEC", C.purple)
     - PLAN -> badge("PLAN", C.blue)
     - DISC -> badge("DISC", C.yellow)
     - VRFY -> badge("VRFY", C.green)
     - PEND -> badge("PEND", C.dimGray)
   - Node name, truncated to fit remaining width
   - Highlight current cursor row with a subtle background or bright text
5. Clear any remaining lines below the tree (in case tree shrunk)
6. Render progress bar on the last line of the area:
   - Format: `[||||||||..] XX% (N/M plans)`
   - Bar width: min(30, width-20) characters
   - Filled portion: `|` repeated proportionally, unfilled: `.`
   - Color: green for filled, dimGray for unfilled
   - Percentage and counts in bright text

**handleKey(name):**
- `"enter"`: Toggle expanded/collapsed on the node at cursorIndex. Only toggle if node has children.
- `"e"`: Expand all nodes recursively (set expanded=true on every node)
- `"w"`: Collapse all nodes recursively (set expanded=false on every non-root node; keep milestones expanded so phases are visible)
- `"j"` or `"down"`: Move cursorIndex down (clamp to visible nodes length - 1). Adjust scrollOffset if cursor moves below viewport.
- `"k"` or `"up"`: Move cursorIndex up (clamp to 0). Adjust scrollOffset if cursor moves above viewport.

**Watcher-triggered re-render:**
When watchPlanning fires its onChange callback:
1. Re-parse tree: `tree = parseGsdTree(planningDir)`
2. Clamp cursorIndex to new visible nodes length
3. If lastRenderArgs is set, call `render(lastRenderArgs...)` to immediately update display

**Edge cases:**
- No .planning/ directory found: render a message "No .planning/ directory found" centered
- Empty ROADMAP.md: render "No phases found" centered
- cursorIndex must be clamped after collapse/expand changes visible node count
